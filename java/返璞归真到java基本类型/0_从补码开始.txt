#从原码、补码、反码开始#

先忘了原码、补码和反码这一大堆的计算。
这些东西大抵是最容易的，一查就知道了。

为什么计算机要用补码存储？
1. 补码可以使得计算机只设计加法运算器，而不用专门设计减法运算器。
2. 补码设计出来只会存储一个0，而不会存储+0和-0

假设计算机的位长是8位：
我们最希望的是用8个0，即0000 0000来表示0
那么用0000 0001 ~ 0111 1111表示1到127的整数
那么1 + (-1) = 0 怎么办？
1: 0000 0001
0: 0000 0000
那么-1怎么表示才好？
很自然，全1咯：1111 1111
这样计算起来， 扔掉溢出位，就是0000 0000

#这样，8位能表示的最大有符号数是多少？#
因为0占了正数的一个位置，让原本0000 0000 到0111 1111，即0到127一共128个数少了一个。
而负数：1000 0000 到 1111 1111,即-128到-1一共128和完整。
所以8位数能表示是-2的7次方到2到7次方减1
