#基本类型的开始#

听说过包装类吗？为什么会存在基本类型和包装类？

java设计之初都是想要everything is Object的。
这样的话，new一个对象要在堆中分配空间。
但是根据我们在以前的处理经验中，有些类型会非常频繁的使用到，在堆中分配释放就显得非常不高效。
所以就出现了java基本类型这个东西，它将变量直接存储在栈中，这样就会非常高效。

但是，这样的话基本类型就不具有对象的特征啊，肿么办？
所以就出现了包装类，它们将这些基本类型包装起来，使得它们具有对象的特征。

这样统一以后，我们在使用集合Collection的时候，就统一只允许使用对象，也方便了后来的集合操作。
比如，当我们向ArrayList、HashMap中放东西时，像int、double这种基本类型是放不进去的。
#容器都只允许装Object。#
要想装基本类型，就必须使用它们对应的包装类。
